## T116 — Accessibility (first automated run)

Built `scripts/audit-a11y.mjs` — Playwright + `@axe-core/playwright` scans 12
primary routes (logins + widget host + admin + agent), runs WCAG 2.0/2.1 A
+ AA rules, prints a summary, exits non-zero on `critical`/`serious` finds,
writes raw JSON to `.audit-output/`.

### First scan — 20 serious violations

| # | Issue | Root cause | Fix |
| --- | --- | --- | --- |
| 1 | `aria-prohibited-attr` on Toaster | `<div aria-live aria-label>` with no `role` | Added `role="status"` to the Toaster container |
| 2 | `color-contrast` 3.74:1 — sidebar muted text on dark rail | `text-rail-foreground/45` too dim | Bumped to `/70` (admin + agent App.tsx) |
| 3 | `color-contrast` 2.39:1 — LanguageToggle in dark sidebar | `text-muted-foreground` is a light gray that fails on the dark rail | Switched LanguageToggle to `text-current/85` so it inherits from its parent context (works in both light login + dark sidebar) |
| 4 | `color-contrast` 1.91:1 — white text on light-pastel avatar bg | Avatar bg lightness 0.78 with chroma 0.10 is too pale for white text on warm hues | Lowered avatar bg to `oklch(0.50 0.17 <hue>)` — passes ≥4.5:1 across every hue stop |
| 5 | `color-contrast` 3.96:1 — primary text in `bg-primary-subtle` Pill | Default primary (oklch 0.58) on subtle teal bg | Pill primary tone now uses `text-[oklch(0.42_0.10_196)]` — same teal family, darker for contrast |
| 6 | `color-contrast` 4.07:1 — "Forgot password?" link on white | 11px medium primary fails 4.5:1 | Bumped to `text-xs font-semibold` + darker oklch — same family, passes |
